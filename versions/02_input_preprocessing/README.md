# 02 Input Preprocessing

Второй срез проекта: страница загрузки данных из `01_basic_page` плюс локальная предобработка входа. Приложение принимает один файл или вставленный текст, проверяет тип и размер, затем строит компактный профиль таблиц или текстовые фрагменты для будущих LLM/графических шагов.

## Запуск

```bash
npm install
npm run dev
```

Страница открывается на `/`.

## Проверки

```bash
npm run typecheck
npm run build
```

Playwright-тесты из корня репозитория:

```powershell
.\tests\02_input_preprocessing\run_02_input_preprocessing_tests.ps1 -BaseUrl http://localhost:3000 -PagePath /
```

## Что добавлено

- CSV/TSV парсинг с определением разделителя.
- XLS/XLSX/ODS обработка, где каждый лист считается отдельным набором данных.
- Нормализация заголовков и базовое определение типов колонок.
- Флаги качества: `id_like`, `high_cardinality`, `mostly_missing`, `pii_like`.
- Нормализация текстов и разбиение длинных текстов на фрагменты до 10 000 символов.
- Предупреждения по данным в отдельном блоке интерфейса.

## Ограничения

- Файлы не отправляются на backend.
- Лимит файла: 10 МБ.
- Лимит текста: 100 000 символов.
- `.doc` принят как формат, но текст из него не извлекается.
- `.docx` принят как формат, но извлечение текста в этой версии не выполняется.
- XLS/XLSX/ODS парсятся в браузере и ограничены памятью устройства.
